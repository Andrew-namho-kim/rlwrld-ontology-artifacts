v0.2 변경 — 수요/공급 축 정리. Account를 사업 → 고객·시장 도메인으로 옮겼습니다. Account는 이제 고객·수요 기업만 뜻하고, 파트너·투자자·컨소시엄사는 Account가 아니라 Organization·Investor로 따로 두어 Partnership.partner·NationalProject.consortium이 참조합니다. 경쟁사의 competesWith도 "모든 경쟁사→RD" 남발을 걷어내고, 유형(CompetitorType)이 실제로 겹치는 사업 라인에만 연결하도록 정정했습니다.

온톨로지는 한 번 만들고 끝이 아니라, 계속 채워지고 갱신되면서 살아 있는 디지털 트윈이 됩니다.

1. 데이터 업데이트 — 살아 있게

  • 주간 딜 동기화 — SI Progress DB에서 stage·금액·확률을 매주 갱신, statusHistory에 변화 누적
  • 정체·변경 감지 — lastActivityAt 21일 초과 = 정체 경보, 지난주 대비 단계 변경 하이라이트
  • 경쟁 인텔 연동 — 이미 도는 경쟁사 Benchmark 파이프라인(주간 리포트)을 Competitor·NewsItem으로 흡수
  • 신선도·출처 — 모든 개체에 sourcedFrom·lastConfirmedAt·confidence를 유지해 "최신·확인된" 정보에 무게
  • 거버넌스 — 월간 스키마 리뷰(답 못 한 역량질문 → 스키마 개정), 주간 신규 편입(경쟁사·고객·용어)

2. 회의록·원천 확장 — 더 넓게

  • 회의록 populate — 큐레이션 77건 등재(외부/전략 45 + 내부 모델·연구 32). 나머지는 CSV 데이터레이어(671건)에 존재, 필요 시 인스턴스화
  • 캘린더 연동 — 구글 캘린더(연결됨)로 CalendarEvent 채우고, 회의록화 여부·아젠다 불명확 미팅 추적
  • 덱 본문 — IR 4.2.1·Business 3.7 덱 본문을 추출해 Narrative로, 내부 진행과 gap 분석
  • 가상랩 심화 — 6개 랩별 연구·논문(Published papers)·서브그룹 진도를 채워 기술 도메인 강화
  • 아직 막힌 것 — 노션 DB 정량 쿼리(플랜 제약, CSV 우회), GitHub 코드 단위(직접 접근 불가 — Platform Team 노션으로 대체)
